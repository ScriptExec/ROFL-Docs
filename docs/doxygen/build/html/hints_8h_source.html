698 /* \brief Allows back-button-press events on Windows Phone to be marked as handled
699 *
700 * Windows Phone devices typically feature a Back button. When pressed,
701 * the OS will emit back-button-press events, which apps are expected to
702 * handle in an appropriate manner. If apps do not explicitly mark these
703 * events as 'Handled', then the OS will invoke its default behavior for
704 * unhandled back-button-press events, which on Windows Phone 8 and 8.1 is to
705 * terminate the app (and attempt to switch to the previous app, or to the
706 * device's home screen).
707 *
708 * Setting the SDL_HINT_WINRT_HANDLE_BACK_BUTTON hint to "1" will cause SDL
709 * to mark back-button-press events as Handled, if and when one is sent to
710 * the app.
711 *
712 * Internally, Windows Phone sends back button events as parameters to
713 * special back-button-press callback functions. Apps that need to respond
714 * to back-button-press events are expected to register one or more
715 * callback functions for such, shortly after being launched (during the
716 * app's initialization phase). After the back button is pressed, the OS
717 * will invoke these callbacks. If the app's callback(s) do not explicitly
718 * mark the event as handled by the time they return, or if the app never
719 * registers one of these callback, the OS will consider the event
720 * un-handled, and it will apply its default back button behavior (terminate
721 * the app).
722 *
723 * SDL registers its own back-button-press callback with the Windows Phone
724 * OS. This callback will emit a pair of SDL key-press events (SDL_KEYDOWN
725 * and SDL_KEYUP), each with a scancode of SDL_SCANCODE_AC_BACK, after which
726 * it will check the contents of the hint, SDL_HINT_WINRT_HANDLE_BACK_BUTTON.
727 * If the hint's value is set to "1", the back button event's Handled
728 * property will get set to 'true'. If the hint's value is set to something
729 * else, or if it is unset, SDL will leave the event's Handled property
730 * alone. (By default, the OS sets this property to 'false', to note.)
731 *
732 * SDL apps can either set SDL_HINT_WINRT_HANDLE_BACK_BUTTON well before a
733 * back button is pressed, or can set it in direct-response to a back button
734 * being pressed.
735 *
736 * In order to get notified when a back button is pressed, SDL apps should
737 * register a callback function with SDL_AddEventWatch(), and have it listen
738 * for SDL_KEYDOWN events that have a scancode of SDL_SCANCODE_AC_BACK.
739 * (Alternatively, SDL_KEYUP events can be listened-for. Listening for
740 * either event type is suitable.) Any value of SDL_HINT_WINRT_HANDLE_BACK_BUTTON
741 * set by such a callback, will be applied to the OS' current
742 * back-button-press event.
743 *
744 * More details on back button behavior in Windows Phone apps can be found
745 * at the following page, on Microsoft's developer site:
746 * http://msdn.microsoft.com/en-us/library/windowsphone/develop/jj247550(v=vs.105).aspx
747 */
